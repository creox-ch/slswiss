### unify-registration (oauth-onboarding) — 🔴 НЕ СДЕЛАНО
Разные пути входа дают разные профили. Google-вход (OAuth) пропускает шаги 2–3 email-регистрации →
у Google-юзеров НЕТ кантона/PLZ/интересов. Кнопка Apple — заглушка (без обработчика).
- Решение (принцип): вход = только аутентификация; кантон/PLZ = общий экран «дозаполни профиль» после ЛЮБОГО входа
- После OAuth, если в профиле нет `canton`/`plz` → показать шаг дозаполнения (кантон, PLZ; интересы опционально) → потом на главную
- Источник истины = обязательные поля (имя, email, кантон, PLZ), а не конкретный UI-путь
- ⚠ Продуктовое решение Ksenia: какие поля обязательны для ВСЕХ (кантон/PLZ — да; интересы — опционально?)
