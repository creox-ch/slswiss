# Contract: Catalog Display (публикация одобренного)

**Спринт:** `catalog-display`
**Дата:** 2026-07-01
**Статус:** ✅ **CLOSED 2026-07-01** — одобренные видны в каталоге/афише, E2E зелёные (БЛОК 16).

---

## Цель

Показать **одобренные** (`status='approved'`) заявки на публичных страницах. Сейчас каталог и афиша
статичные (захардкожены), из БД не читают — поэтому approve «уходит в никуда». Подключить их к
`services`/`events` и рендерить одобренные записи.

## Не входит в спринт

- Детальная страница карточки бизнеса/события (отдельно)
- Загрузка картинок/логотипов (media-uploads)
- Рейтинги/отзывы
- Сложная серверная фильтрация/пагинация (MVP: клиентский рендер всех approved)
- Биржа (`tasks`) — только каталог и афиша

## Точки внедрения (из реального HTML)

- Каталог: контейнер `#biz-grid` (сейчас статичная заглушка «Каталог пополняется»)
- Афиша: панель событий `#afisha-ev` (есть empty-state `events-empty-afisha`)
- Загрузка — при `goPage('catalog')` / `goPage('afisha')` (по аналогии с `updateEmptyStates`)

## Схема БД

- Отдельная не нужна. RLS уже пускает anon к `status='approved'` (миграции forms + moderation).

---

## Сценарии — Каталог

- [ ] При открытии каталога грузятся `services` где `status='approved'` → рендерятся карточки в `#biz-grid`
- [ ] Карточка показывает: название, категорию, кантон, описание, контакты (сайт/telegram, если есть)
- [ ] Заявки `pending_moderation` / `rejected` НЕ показываются (проверка на уровне RLS + запрос)
- [ ] Нет approved → показывается empty-state «Каталог пополняется» (как сейчас)
- [ ] Одобрил в модерации → появилось в каталоге (после перезахода/перезагрузки)

## Сценарии — Афиша

- [ ] При открытии афиши грузятся `events` где `status='approved'` → рендерятся в списке событий
- [ ] Показывает: название, дату, кантон, описание
- [ ] pending/rejected НЕ показываются
- [ ] Нет approved → empty-state

## Технические инварианты

- [ ] Только anon-public key (approved и так публичны)
- [ ] Экранирование пользовательского текста (XSS) — как в модерации (`_modEsc`)
- [ ] `node --check` (в CI) — существующие тесты не сломаны
- [ ] Ошибка загрузки из БД → не ломает страницу, показывается заглушка

## Тесты (TDD — `tests/11-catalog-display.spec.js`)

- [ ] seed approved `services` → открыть каталог → карточка видна (по названию)
- [ ] seed pending `services` → открыть каталог → НЕ видна
- [ ] seed approved `events` → открыть афишу → видно
- [ ] очистка seed через service_role
