# Contract: Mobile Adaptive

**Спринт:** `mobile-adaptive`
**Дата:** 2026-07-01
**Статус:** `draft` (от Planner, ждёт подтверждения Ksenia)

---

## Цель

Сделать сайт пригодным для просмотра с телефона. Сейчас в `index.html` **ноль `@media`-запросов** —
вся вёрстка на фиксированных desktop-сетках, на мобильной ширине (≤390px) ничего не перестраивается:
шапка с 7 пунктами не влезает, hero и многоколоночные сетки уезжают/сжимаются. Добавляем один
адаптивный слой (`@media (max-width:768px)`, при необходимости 480px) для всех основных экранов + гамбургер-меню.

## Не входит в спринт

- Редизайн/новая визуальная концепция — только адаптив существующей вёрстки.
- Планшетные тонкости (768–1024) сверх базового брейкпоинта.
- Изменение контента/логики страниц.
- PWA, offline, тач-жесты.

## Зависимости

- **Зависит от:** ничего (чистый CSS + маленький JS для меню).
- **Блокирует:** запуск для тест-группы 300 (люди зайдут с телефонов).

## Зафиксированные продуктовые решения

- ✅ **Мобильная навигация — гамбургер-меню** (☰ → выпадающая панель). — Ksenia, 2026-07-01
- ✅ **Объём — все основные экраны** (Главная, Афиша, Кантоны, Каталог, Биржа, Статьи, Профиль, футер). — Ksenia, 2026-07-01

---

## Пользовательские сценарии

- [ ] На ширине ≤480px нет горизонтального скролла (страница не «уезжает» вбок) на всех основных экранах
- [ ] Шапка: пункты меню скрыты, видна кнопка ☰; тап по ☰ открывает панель со всеми пунктами
- [ ] Тап по пункту меню → переход на страницу И меню закрывается
- [ ] Hero: заголовок/текст/кнопки и блок статистики идут одной колонкой (не 2), читаемо
- [ ] Каталог/Афиша/Кантоны/Биржа/Статьи: карточки идут 1–2 в ряд, не сплющены
- [ ] Профиль: боковое меню и контент — одной колонкой
- [ ] Кнопки «Войти»/«Вступить» остаются доступны в шапке на мобиле

## Edge cases

- [ ] Гамбургер-меню закрывается при навигации (не остаётся открытым поверх новой страницы)
- [ ] Инлайновые `grid-template-columns` в HTML переопределяются (через `!important` в media-блоке)
- [ ] На desktop (>768px) поведение НЕ меняется: ☰ скрыта, меню горизонтальное как раньше
- [ ] Модалки (вход/регистрация/дозаполнение/формы) остаются в пределах экрана (уже `max-width:9xvw`)

## UI-инварианты

- [ ] Брейкпоинт `@media (max-width:768px)` (осн.) добавлен перед `</style>` — переопределяет базовые правила
- [ ] `.nav-burger` виден только на мобиле; на desktop `display:none`
- [ ] `document.documentElement.scrollWidth <= clientWidth` при 390px на всех страницах (нет overflow)
- [ ] Целевые тап-зоны пунктов меню ≥ 40px по высоте
- [ ] Текст hero-заголовка уменьшен, не обрезается

## Технические инварианты

- [ ] Нет ошибок в `console.error` при открытии/навигации на мобиле
- [ ] Существующие Playwright-тесты (desktop, блоки 01–20) не сломаны
- [ ] `node --check` на главном script зелёный (проверка через CI — локально node нет)
- [ ] `index.html` вырос не более чем на ~6KB

## Решения и заметки

- **2026-07-01 (Planner):** корень — 0 `@media` в файле; viewport-meta уже есть. Правим CSS-слоем, не трогая разметку (кроме одной кнопки ☰ в `<nav>` и `toggleNav/closeNav`).
- **2026-07-01 (Planner):** много сеток заданы инлайном в HTML (`#biz-grid`, `#cantons-grid` и др.) → в media-блоке использовать `!important`, иначе инлайн-стиль победит.
- **2026-07-01 (Planner):** Playwright реально эмулирует мобильный viewport (в CI проект `chromium-desktop` — в новом спец-файле тестов зададим `test.use({ viewport })`). Chrome-MCP из Cowork рендерит в фикс. desktop-ширине, им мобилку не проверить — только Playwright/реальный телефон.
