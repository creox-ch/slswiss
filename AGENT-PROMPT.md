# Agent Prompt Template

Copy this into Cowork when starting a new sprint. Replace `<placeholders>`.

---

## Template (copy below)

```
Задача: реализовать спринт `<sprint-name>` согласно контракту.

Шаг 0 — прочитай CLAUDE.md в корне репо. Там правила работы, real selectors, типичные баги, JS validation pattern. Не пропускай.

Шаг 1 — прочитай контракт: `contracts/<sprint-name>.md` в Project knowledge.
Это ~N проверяемых утверждений. Контракт = истина. Не выходи за scope.
Если контракт кажется неправильным или неполным — СТОП, спроси Ksenia, не додумывай.

Шаг 2 — прочитай `IVANNA-NEXT-SPRINT.md` в Project knowledge для конкретных подзадач и порядка.

Шаг 3 — реализуй пункт за пунктом из контракта. Маленькие коммиты с понятными сообщениями.
Перед каждым коммитом:
- Запусти node --check на главном script (паттерн в CLAUDE.md)
- Если задеваешь что-то с тестами — прогон локально через --ui

Шаг 4 — после каждого push следи за CI:
- gh run list --limit 1 --workflow=test.yml
- Если красный: gh run view <id> --log-failed
- Классифицируй: site bug / test bug / flake
- Фикси и push снова

Шаг 5 — когда все пункты контракта зелёные:
- Обнови BACKLOG.md (спринт → done, дата)
- Если архитектура поменялась — обнови slswiss-architecture.md
- Сообщи мне, я скажу Ksenia для финального review

Ограничения:
- НЕ расширяй scope. Всё что не в контракте → BACKLOG как новый пункт
- НЕ деплеть тесты ради зелёного CI
- НЕ пушь если node --check падает
- НЕ пушь если предыдущий CI красный и причина не пофикшена

Открытые вопросы для этого спринта:
- <блокер 1, если есть>
- <блокер 2, если есть>

Начинай с шага 0.
```

---

## Per-sprint customizations

### registration-fix (текущий)

Замени `<sprint-name>` на `registration-fix`, `~N` на `~38`.

Блокеры:
- Custom SMTP провайдер — по умолчанию Resend (3000 писем/мес бесплатно). Если есть сильные причины выбрать SendGrid / Google Workspace SMTP — обсуди до настройки.
- DNS записи в Hostpoint требуют доступа Kseniia.

Конкретные подзадачи — в `IVANNA-NEXT-SPRINT.md`.

### Будущие спринты

Когда контракт следующего спринта готов и согласован, ты или planner-чат обновляют `IVANNA-NEXT-SPRINT.md`. Затем копируешь шаблон выше, подставляешь имя спринта и блокеры, и запускаешь.

---

## Анти-паттерны (чего не должно быть в промпте)

- ❌ «Сделай хорошо» — расплывчатое, агент сам решает что значит «хорошо»
- ❌ Полный список требований внутри промпта — это есть в контракте, не дублируй
- ❌ «И вообще ещё пофикси X, Y, Z» — расширение scope, отдельный спринт
- ❌ «Если что — решай сам» — для продуктовых решений всегда стоп и спрос

---

## Что делать когда агент завис

Если в Cowork агент ходит кругами или делает что-то странное:

1. Останови
2. Прочитай его последние действия — где сбился
3. Возможные причины:
   - CLAUDE.md устарел — обнови
   - Контракт расплывчатый — уточни и финализируй заново
   - Блокер не указан в промпте — добавь
   - Скоуп слишком большой — раздроби на меньший спринт
4. Перезапусти с уточнённым промптом

---

## Эволюция этого шаблона

После каждого спринта смотри: что не сработало в промпте, что добавить, что убрать. Обнови этот файл. Через 3-4 спринта он стабилизируется.
