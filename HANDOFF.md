## Реальная структура сайта (важно для селекторов)

- Модалка логина: `#m-login`
  - Email: `#login-email-input`
  - Пароль: `#login-pwd-input`
  - Submit: getByRole('button', { name: 'Войти', exact: true }).last()
- Модалка регистрации: `#m-reg`
  - Шаги в контейнерах: `#rs1`, `#rs2`, `#rs3`
  - Поля Шаг 1 — по плейсхолдеру: `getByPlaceholder('Имя')`, etc.
  - Шаг 2 — `#rs2 select` (Kanton), `#rs2 input` (PLZ)
- Навигация: `nav li:has-text("Кантоны")` (это `<li>`, не `<a>`!)
- Триггер логина в шапке: `button.btn-login`
- Триггер регистрации в шапке: `button.btn-join:has-text("Вступить")`
- Кнопка `+ Добавить в каталог`: **верхняя в баннере сломана**, рабочая — нижняя в пустом состоянии (`.last()`)
- baseURL: `https://creox-ch.github.io/slswiss/` → `page.goto('')` (НЕ `'/'`)

## Новая задача (текст из чата)

Тесты для функции «Редактировать профиль» — `tests/05-profile-edit.spec.js`.

**Готовый ТЗ:** `test_profile_edit.md` (от пользователя, 10 блоков, 50+ кейсов)

**Черновик спека от пользователя:** `05-profile-edit.spec.js` — есть проблемы:
- 5.1 без ассертов — всегда зелёный
- ID `#prof-first`, `#prof-last`, `#prof-canton` и т.д. — **угаданные**, надо проверить в реальном HTML
- Тексты `text=Настройки профиля`, `text=Профиль обновлён` — тоже угаданные
- `.catch(() => {})` глотает ошибки молча
- 10+ `waitForTimeout(500)` без необходимости
- Дубль логики «открыть настройки» в каждом тесте — вынести в `openProfileSettings` helper
- 5.10 после `page.reload()` сессия может слететь
- 5.11 race condition: кнопка disabled проверяется после клика, к этому моменту может быть уже enabled

## Хорошие практики из этого цикла

1. **Триаж падений** — категоризировать: тест/сайт/устарел. Не лезть в код пока не классифицировал.
2. **Page snapshot из отчёта** ценнее догадок. Всегда читай Error → Page snapshot → Screenshot, в таком порядке.
3. **Реальные ID > угаданные.** Открыть F12 → Elements → скопировать. Никогда не писать селектор по памяти.
4. **`:visible` фильтр** когда дубли в DOM (например логин и регистрация формы с одинаковыми плейсхолдерами).
5. **`.last()` или scope** когда триггер и submit имеют одинаковый текст.
6. **Никогда не пушь красный тест в main** — либо чини, либо `.skip()` с TODO.
7. **Локальный прогон через `--ui` режим** — итерируй секундами, не ждя CI.
